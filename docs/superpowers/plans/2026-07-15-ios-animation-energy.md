# iOS Animation Energy Optimization Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:test-driven-development for the behavior change. This plan is tightly coupled across one standalone HTML artifact and its deploy copy, so it is executed as one implementation stream with an independent final review.

**Goal:** Make mobile weather animation energy-bounded, ensure dry weather performs no Canvas work, and publish the optimized artifact.

**Architecture:** `tide-redesign.html` remains the editable source. One central runtime coordinator controls a timer-paced Canvas rain engine and CSS pause state; `tide-deploy/index.html` is generated by exact source synchronization and guarded by validation.

**Tech Stack:** Standalone HTML/CSS/JavaScript, Canvas 2D, Node.js validation, Playwright/Chromium diagnostic profiling, Cloudflare Workers static assets.

## Global Constraints

- Preserve the current UI, weather visuals, localization, tide calculations, and network integrations.
- Do not add dependencies or credentials.
- Mobile default is 20 FPS, DPR 1.0, storm drop scale 0.30.
- Low tier is 12 FPS, DPR 1.0, storm drop scale 0.20.
- Rain Canvas runs only for `rain` and `storm`.
- No independent full-rate animation governor is allowed.
- `tide-redesign.html` and `tide-deploy/index.html` must be byte-identical.
- Finish with one reviewed commit and push to `origin/main`, as explicitly requested by the user.

### Task 2: Implement the energy-budgeted renderer

**Files:**
- Modify: `tide-redesign.html`
- Replace mechanically from source: `tide-deploy/index.html`

**Interfaces:**
- Consumes: button state, `document.hidden`, `IntersectionObserver`, reduced-motion, quality tier, weather state.
- Produces: `shouldAnimate()`, `shouldRunRain()`, `syncRuntime()`, timer-paced `RainEngine`, and byte-identical deploy HTML.

- [ ] **Step 1: Simplify CSS/SVG rendering**

  Remove moving SVG blur/filter attributes, dial-wide shadow, animated beacon geometry, and sunny blur. Restrict cloud animation to visible weather. Enforce medium/low scene budgets from the design.

- [ ] **Step 2: Replace the RainEngine scheduler**

  Maintain one `frameTimer` and one pending `rafId`. Schedule the next frame with `setTimeout(() => requestAnimationFrame(render), 1000 / quality.fps)`. Refuse `start()` when intensity is zero. `setWeather()` stops immediately for dry weather.

- [ ] **Step 3: Remove the independent governor**

  Delete `MotionGovernor`. Count late frames in `render()` and request at most one quality downgrade per 120 rain frames through `WeatherQuality.downgrade()`.

- [ ] **Step 4: Centralize lifecycle state**

  Implement `shouldAnimate()`, `shouldRunRain()`, and `syncRuntime()`. Persist `pageVisible` and `appVisible`. Route button, weather, `visibilitychange`, `pagehide`, `pageshow`, observer, and quality changes through `syncRuntime()`.

- [ ] **Step 5: Fix adjacent lifecycle defects**

  Attach the bubble iteration listener to the dial rather than the unrelated weather-stage sibling. Remove the duplicate `RainEngine.setTier()` call from the quality listener.

- [ ] **Step 6: Update release identity and synchronize deploy**

  Set `APP_VERSION` to `2026.07.15.1`, then copy `tide-redesign.html` exactly to `tide-deploy/index.html`.

- [ ] **Step 7: Verify GREEN**

  Run `npm run check`.

  Expected: `Deploy files look ready.` and exit code 0.
